# 2026年2月22日 — 每日备忘录

## 今日完成工作

### 1. 修复关键 Bug：`projectToWorld()` 相机坐标符号
ARKit 相机坐标系为 **+X 右, +Y 上, -Z 前方**。原实现符号错误：

```swift
// 修复前（错误）：
let y_cam = (Float(by) - cy) * d / fy   // 缺少负号
let z_cam = d                              // 应为 -d

// 修复后（正确）：
let y_cam = -d * (Float(by) - cy) / fy    // 图像Y向下 vs 相机Y向上
let z_cam = -d                              // 相机朝 -Z 方向看
```

**影响：** 所有世界坐标 Y 值都是反的 —— 地面被当成天花板，高度分类完全错误，坡道检测产生误报（如墙壁被识别为坡道），手机倾斜角度无法正确影响障碍物检测。

### 2. 修复关键 Bug：距离参考点错误
所有 `hdist` 计算使用的是到**世界原点**的距离，而非到**相机位置**的距离：

```swift
// 修复前（错误）：到世界原点的距离
let hdist = sqrtf(pt.x * pt.x + pt.z * pt.z)

// 修复后（正确）：到相机位置的距离
let dx = pt.x - cameraPos.x
let dz = pt.z - cameraPos.z
let hdist = sqrtf(dx * dx + dz * dz)
```

**影响：** 修复 Bug 1 后，一切都变成了 P5（畅通），因为用户走离 ARKit 会话起点后，所有障碍物距离都显得很远。这个 bug 之前被 Bug 1 掩盖（错误的符号在原点附近时距离"凑巧"不太离谱）。

**修复涉及：** `computeFreeSpaceMap()`、`detectStairs()`、`detectSlope()`

### 3. 改善坡道检测鲁棒性
- 最少地面点数：20 → **30**
- 新增**最小距离跨度检查**：地面点水平距离跨度 ≥ 1.0m（防止面对墙壁时仅有少量墙脚地面点导致误报）
- 新增 **R² > 0.5** 检验：线性回归必须能良好拟合数据才报告坡道（防止噪声引起的误报）

### 4. 新增相机俯仰角 Debug 日志
- 从 transform 矩阵计算相机俯仰角：`asin(forward.y)` 转为角度
- 在 `[B2]` console 输出中新增 `pitch=XX°`（0° = 水平，负值 = 向下看）
- 便于在真机上验证手机倾斜角度是否正确影响障碍物分类
